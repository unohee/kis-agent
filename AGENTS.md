# pykis 개발 가이드

## 프로젝트 개요
pykis는 한국투자증권 API를 Python으로 쉽게 사용할 수 있도록 래핑한 라이브러리입니다. 본 프로젝트는 pip를 통해 설치하여 사용할 수 있는 형태로 개발되고 있습니다.

## 핵심 컴포넌트
- `Agent`: 메인 래퍼 클래스 (Facade 패턴)
- `Client`: API 통신을 담당하는 기본 클라이언트
- `StockAPI`: 주식 관련 API 기능
- `ProgramTradeAPI`: 프로그램 매매 관련 API 기능
- `ConditionAPI`: 조건검색 관련 API 기능
- `AccountAPI`: 계좌 관련 API 기능

## 주요 기능
1. 계좌 관리
   - 계좌 잔고, 주문 가능 금액, 총 자산 조회
   - 주문 내역 및 체결 내역 조회
   - 신용 주문, 주문 정정/취소, 예약 주문
2. 주식 시세 조회
   - 국내주식: 현재가, 일별시세, 호가정보, 분봉 데이터
   - 해외주식: 기간별시세, 뉴스, 권리정보
   - 채권: 기간별시세
3. 시장 분석
   - 체결강도 순위 (올바른 API 적용)
   - 등락률 순위 (국내주식 등락률 순위 API)
   - 거래량 순위
   - 투자자별 매매 동향
   - 증권사별 투자의견
4. 재무정보
   - 손익계산서
   - 재무비율
   - 성장성/안정성 지표
5. 프로그램 매매 정보 조회
   - 종목별 프로그램 매매 추이
   - 시장 전체 프로그램 매매 현황
   - 독립적인 분석 도구 (`examples/program_trade_analysis.py`)
6. 조건검색식 종목 조회 (통일된 API 방식)
7. 휴장일 정보 조회 (직접 API 접근 + 편의 메서드)
8. 실시간 시세 조회

## 최근 주요 개선사항 (v0.1.5)
1. **조건검색 API 통일**
   - 모든 조건검색 호출이 `condition.py`의 정확한 방식(`tr_id="HHKST03900400"`) 사용
   - `user_id="unohee"` 매개변수 통일 사용
   - `rt_cd='1'` 응답 정상 처리 개선
   - Facade 패턴 일관성 강화: Agent를 통한 통합 접근

2. **휴장일 기능 추가**
   - `StockAPI`에 `get_holiday_info()` 및 `is_holiday()` 메서드 추가
   - Agent 클래스에 휴장일 기능 통합
   - 직접 API 접근과 편의 메서드 제공
   - 기준일 계산 로직 개선 및 재시도 로직 포함

3. **아키텍처 일관성 개선**
   - Facade 패턴 완전 준수: 모든 기능을 Agent를 통해 접근
   - 내부 API 클래스는 캡슐화하되 적절한 책임 분리 유지
   - 사용자 인터페이스 통일

3. **분석 기능 독립화**

## 개발 환경 설정
1. Python 3.8 이상
2. 가상환경 설정 (venv)
3. credit/kis_devlp.yaml 파일에 API 인증 정보 설정

## 테스트
- **종합 테스트**: `examples/pykis.ipynb` (50+ 메서드 포괄 테스트, 모듈 재로드 기능 포함)
- 통합 테스트: `kis_integration_test.py`
- 조건검색 테스트: `test_condition_stocks.py` (API 통일 완료)
- 프로그램매매 테스트: `test_program_trade.py`
- 시장정보 테스트: `test_market.py`, `test_market_api.py`

## 주의사항
- API 호출 시 실제 주문이 발생할 수 있으므로 테스트 시 주의
- 인증 정보는 반드시 안전하게 관리
- API 호출 제한에 주의
- 에러 발생 시 로그 확인 필수
- 한국투자증권 OpenAPI 정책 변경에 따라 일부 기능이 영향받을 수 있음
- 조건검색 API는 반드시 올바른 `user_id` 설정 필요

## 개발 가이드라인
0. 해당 저장소를 개발하기 위해서는 가상환경 활성화가 필수적이다. 현 시스템에선 가상환경 활성화 alias인 'venv'를 입력하거나 source ~/RTX_ENV/bin/activate 입력하고 작업을 시작할 것.
1. **ContextStrictMode**: 지정된 컨텍스트 블록 외에는 수정 금지
2. **KoreanResponseOnly**: 모든 응답은 한국어로 작성
3. **NoNamingAlteration**: 기존 변수명/함수명 유지
4. **NoSilentInference**: 변경 이유를 한국어 주석으로 명시
5. **MinimalTouch**: 필요한 최소한의 수정만 실행
6. 모든 API 메서드에 docstring 추가 (목적, 파라미터, 반환값, 예외 처리)
7. 로깅 메시지는 반드시 한국어로 작성
8. 타입 힌트 사용 필수
9. 테스트 코드 작성 및 검증 필수

## 아키텍처 원칙
- **Facade 패턴 준수**: Agent 클래스를 통한 모든 기능 접근
- **API 래퍼와 분석 로직 분리**: 순수한 API 호출과 비즈니스 로직 분리
- **단일 책임 원칙**: 각 모듈은 명확한 단일 책임을 가짐
- **올바른 API 사용**: 한국투자증권 공식 API 문서에 따른 정확한 구현
- **일관된 매개변수**: 조건검색 등에서 통일된 매개변수 사용
- **에러 처리 강화**: 모든 API 호출에 대한 적절한 예외 처리
- **로깅 표준화**: 일관된 한국어 로깅 메시지 사용
